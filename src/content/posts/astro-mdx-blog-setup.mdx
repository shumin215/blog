## 왜 Astro인가?

블로그 프레임워크를 고를 때 여러 선택지가 있었습니다.
Next.js, Gatsby, Hugo… 그중 **Astro**를 선택한 이유는 단순합니다.

> 블로그는 **읽히는 것**이 가장 중요하다.

Astro는 기본적으로 JavaScript를 최소화해 정적 HTML을 생성합니다.
덕분에 Lighthouse 점수가 높고 로딩이 빠릅니다.

<Callout type="warn">
  Astro v6부터 content config 파일 위치가 `src/content/config.ts`에서 `src/content.config.ts`로 변경됐습니다.
</Callout>

## MDX의 장점

일반 Markdown에서는 React 컴포넌트를 쓸 수 없지만,
MDX를 사용하면 이런 것들이 가능합니다.

```mdx
import Callout from '@components/Callout.astro';

<Callout type="info">
  이 부분은 중요한 내용입니다!
</Callout>
```

## 프로젝트 구조

```
src/
├── content/
│   ├── config.ts        # 스키마 정의
│   └── posts/           # .mdx 파일들
├── components/
├── layouts/
└── pages/
```

## 설치 과정 요약

```bash
npm create astro@latest
npx astro add tailwind mdx sitemap
npm install -D @tailwindcss/typography prettier-plugin-astro
```

다음 포스트에서는 Docker로 배포하는 방법을 다루겠습니다.
